# NPS 管理端 UI 设计需求文档

> 本文档描述 **Web 管理界面** 的视觉主题、布局结构与交互表现，供整体 UI 重构与验收使用。  
> **默认语言：简体中文。**  
> **设计参照：飞书（Feishu / Lark）产品设计语言** —— 高效协作向的企业后台：**主题配色须采用飞书主色蓝与中性灰阶体系**（见 **§2.1**），清晰信息层级、克制留白，与飞书办公套件在 **色彩 Token、组件密度、图标体系** 上保持一致（不照搬业务，仅对齐设计语言）。

---

## 1. 设计目标与原则

| 原则 | 说明 |
|------|------|
| 可读优先 | 数据密集场景（列表、监控）保证字号、行高、对比度符合长时间运维阅读。 |
| 层级清晰 | 背景 / 表面 / 悬浮 / 强调，四层以内，避免同屏过多强对比色块。 |
| 企业 / 集团化 | 视觉与信息结构体现 **组织级管控**：工作台总览、分组树、权限与归属清晰，避免「单机工具感」。 |
| 触控友好 | 主要操作区点击热区不小于 44×44px（桌面端可适当收紧，但需保留舒适间距）。 |
| 一致性 | 同类页面（列表、表单、详情）采用同一套布局栅格与组件形态。 |
| 中文默认 | 所有文案、日期、数字格式默认按简体中文习惯；预留英文等语言切换，不改变布局结构。 |

## 2. 主题与视觉规范（飞书向）

### 2.1 主题配色（须按飞书色系，禁止自造主色）

本管理端 **整体主题** 须对齐 **飞书（Feishu / Lark）产品默认浅色主题** 的配色逻辑与气质，与飞书客户端、管理后台、开放平台文档等 **同一套「蓝 + 灰」体系**。  
实现时优先采用 **飞书设计体系中的 Token / 色阶命名**（如 `blue`、`neutral`、`N*`、`grey` 等；具体以 **飞书官方设计资源 / LDS** 为准），下表为 **等效 HEX 参考**，供无 Token 环境时对照，**不得**改用橙、紫等非飞书主色作为全局主色。

#### 2.1.1 品牌主色（Primary / Blue）

飞书全局强调色为 **蓝色系**，主操作、链接、选中、图表主系列均从 **同一主色** 衍生，保持 **色相一致**。

| 用途 | 参考色值 | 说明 |
|------|-----------|------|
| **主色默认** | `#3370FF` | 主按钮填充、关键链接、选中文字、图标强调；与飞书产品主蓝一致 |
| **悬停（Hover）** | `#4E83FD` | 按钮 / 文字链接悬停，略提亮 |
| **按下（Active）** | `#245BDB` | 按钮按下，略压暗 |
| **浅色底（选中背景）** | `#E8F0FF` | 侧栏选中项、表格行弱选中、树节点选中底 |
| **更浅浅底** | `#F2F8FF` | 可选：斑马区隔、hover 行底 |
| **禁用（主色控件）** | `#3370FF` @ **38%～45% 不透明度** 或 **#8FADFF** | 禁用主按钮，勿用灰冒充蓝 |

**禁止**：用 `#FF6700`、`#722ED1` 等作为 **全局主色**；局部标签小面积语义色除外（见 2.1.4）。

#### 2.1.2 中性色（背景 / 边框 / 文字）

与飞书 B 端界面一致：**浅灰背景 + 白卡片 + 深灰字**，层级靠 **对比度** 区分，不靠彩色背景铺满。

| 用途 | 参考色值 | 说明 |
|------|-----------|------|
| **页面背景** | `#F5F6F7` | 主内容区背后一层；可浅于 `#F2F3F5` 二选一，全站统一即可 |
| **表面 / 卡片** | `#FFFFFF` | 卡片、抽屉、下拉面板内容区 |
| **一级边框** | `#DEE0E3` | 卡片外框、模块分割 |
| **二级边框 / 分割线** | `#E5E6EB` | 列表行、表头底线 |
| **文字主色** | `#1F2329` | 标题、正文主文 |
| **文字次色** | `#646A73` | 次要说明、表头、图标默认灰 |
| **文字辅助** | `#8F959E` | 占位符、更弱说明 |
| **文字禁用** | `#BBBFC4` | 禁用文案、不可用项 |

#### 2.1.3 填充与层级（中性浅底）

| 用途 | 参考色值 |
|------|-----------|
| **输入框背景** | `#FFFFFF` 或 `#F5F6F7`（弱灰底输入，二选一统一） |
| **悬停行背景** | `#F5F6F7` 或 `#F2F3F5` |
| **表头背景** | `#FAFBFC` 或 `#F5F6F7` |

#### 2.1.4 语义色（状态 / 反馈）

与飞书系 **成功绿、警示橙、错误红** 的语义习惯对齐，**仅用于**状态标签、结果提示、校验错误，**面积小于主色**。

| 语义 | 参考色值 | 典型用途 |
|------|-----------|----------|
| **成功** | `#00B42A` | 成功提示、在线、已启用（绿系标签底可用 `#E8FFEA` + 字 `#00B42A`） |
| **警告** | `#FF8800` | 告警条、需注意（浅底可用 `#FFF7E8`） |
| **错误 / 危险** | `#F54A45` | 表单错误、删除、失败（浅底可用 `#FFECE8`） |
| **信息** | `#3370FF` | 与主色一致的信息条；或浅底 `#E8F0FF` |

#### 2.1.5 图表配色

- **主数据系列**：`#3370FF`；次系列可用 **同系浅蓝** `#85A9FF`、`#BEDAFF` 或中性 `#8F959E`，保持 **同一图表内色相不杂乱**。  
- **饼图 / 环图**：优先 **蓝系渐变 + 中性灰** 区分块，避免彩虹色块与飞书后台气质冲突。

#### 2.1.6 落地要求（验收）

- 设计稿（Figma）须建立 **颜色样式（Color Styles）**，命名与 **飞书 Token 语义** 对齐（如 Primary / TextTitle / Border / BgPage）。  
- **开发**优先使用 **CSS 变量** 引用同一套 Token，禁止页面内散落硬编码相近蓝。  
- **深色模式**：若二期做，须单独按 **飞书深色主题** 色板另起一套，不与浅色混用。

### 2.2 字体与排版

- **字体家族**：无衬线 UI 字体栈；中文优先 **苹方 / 思源黑体 / 阿里巴巴普惠体** 等与飞书气质接近的字体；数字与西文与中文基线对齐。
- **字重**：标题可用 Medium / Semibold；正文 Regular；标签与辅助文案避免过多字重档位（通常 2～3 档即可）。
- **字号阶梯**：页面标题 > 区块标题 > 表头 / 表单标签 > 正文 > 辅助说明；仪表盘 KPI 数字可单独一档 **略大、字重略高**。
- **行高**：正文不小于 **1.5**；表格与密集数据区在可读前提下可适当收紧，但不得挤压为多行难辨。
- **数字**：流量、端口、版本号等建议 **等宽或 Tabular**，便于列表纵向对齐扫读。

### 2.3 圆角、阴影与层次

- **圆角**：飞书向偏 **克制圆角** —— 大卡片、弹窗 **8～12px**；按钮、输入框、小标签 **4～8px**；同一界面避免混用多组无关联圆角。
- **阴影**：以 **浅扩散 + 低透明度** 为主（如 `0 2px 8px rgba(0,0,0,.08)` 量级）；下拉、抽屉可略加强；避免厚重拟物阴影。
- **层次**：背景灰 → 白卡片 → 悬浮层 → 模态；用 **边框 + 浅阴影** 二选一或弱组合即可，忌层层叠加重阴影。

### 2.4 图标系统（功能定制 + 选中 / 未选中态）

**总体要求**

- **禁止**用「一套通用线性图标随意对应功能」：须 **按功能语义单独绘制**（见下清单），保证 **造型可辨认、风格统一**（同一套网格、笔画规则、端点与圆角处理一致）。
- **风格**：建议 **线性为主、线宽统一**（如 1.5px～2px@1x）；可与飞书系图标保持 **几何、简洁、少装饰** 的倾向；如需面性图标，须与线性套系同一套图形基因。
- **尺寸**：导航与列表主图标 **20～24px** 栅格；表格行内操作 **16～20px**；需导出 **1x / 2x** 或 SVG。

**必选：选中态与未选中态（双态）**

以下场景 **必须** 提供 **未选中 + 选中** 两套视觉（或同一资源内可切换状态），交互上随状态即时切换，无闪烁错位：

| 场景 | 未选中 | 选中 |
|------|--------|------|
| **侧栏主导航项** | 图标 + 文字 **次色**（见 **§2.1.2** `#646A73`）；背景透明或与侧栏底同色 | 图标 + 文字 **主色** `#3370FF`，背景 **浅蓝底** `#E8F0FF`（见 **§2.1.1**）或左侧主色条 |
| **顶栏 Tab / 分段控件**（若有） | 默认文字与图标弱化 | 当前项主色 + 可选下划线或胶囊底 |
| **表格行勾选**（若有） | 未勾选框样式 | 勾选后主色填充或主色勾 |
| **隧道类型等筛选 Chip**（若有） | 线框或浅灰底 | 主色浅底 + 主色描边或文字 |

**功能图标清单（须专项绘制，各 2 态）**

每个条目均需 **独立图形**，并与业务含义强相关；**禁止**复用「齿轮表示一切设置」类偷懒映射。

| 功能 | 图标语义方向（供插画师发挥，需评审） |
|------|--------------------------------------|
| 仪表盘 | 总览、数据看板、仪表盘 |
| 客户端 | 终端、设备、连接管理 |
| 域名解析 | 域名、地球、HTTP 路由 |
| TCP | 双向箭头、链路 |
| UDP | 数据报、非连接 |
| HTTP 代理 | 代理、网关 |
| SOCKS5 | 代理层、隧道套接字 |
| 秘密通道 | 隐身、密钥、通道 |
| P2P | 点对点、直连 |
| 文件服务 | 文件夹、静态资源 |
| 客户端分组（树节点） | 组织、树形、文件夹叠层 |
| 帮助 | 问号、文档、指南 |

**状态类（在线/离线、启用/关闭等）**

- 仍以 **胶囊标签 + 语义色** 为主；若配图标，须 **独立小图标**（如圆点、插拔），与上表「功能图标」区分层级，避免混淆。

**交付物**

- 每枚图标 **SVG**（或 PDF 矢量）+ **未选中 / 选中** 两态切图命名规范（如 `nav-dashboard.svg`、`nav-dashboard-active.svg`）。  
- **图标使用说明一页**：网格、线宽、安全区、禁止拉伸变形规则。

## 3. 整体布局架构

### 3.1 框架（登录后）

- **结构**：左侧 **固定导航栏** + 右侧 **顶栏** + **主内容区** + 可选 **底栏**。
- **侧栏宽度**：折叠前约 220～260px；支持收拢为图标-only 窄栏，主内容区自适应。
- **顶栏**：左侧可留面包屑或当前页标题；右侧为语言切换、用户身份、退出等，信息密度对齐 **飞书后台**（简洁、少装饰、操作集中右侧）。
- **主内容区**：默认水平居中最大宽度（如 1200～1440px）或全宽 + 内边距；**客户端列表页例外**：采用 **左侧分组树 + 右侧列表** 全宽双栏（见 **4.3**），与集团化工作台一致。

### 3.2 导航信息架构（顺序可微调，需保持分组清晰）

1. **仪表盘** — 总览与监控  
2. **客户端** — 客户端列表与维护  
3. **域名解析** — 域名规则  
4. **隧道类型** — TCP / UDP / HTTP 代理 / SOCKS5 / 秘密通道 / P2P / 文件服务（可收为「隧道」二级分组）  
5. **帮助** — 可外链文档，建议新标签打开  

### 3.3 登录前（总体）

- 独立全屏入口，与登录后工作台形成 **「品牌门户 → 集团工作台」** 的体验递进。
- 若开放注册：登录与注册页 **视觉体系统一**（同一套顶栏、背景、主按钮与页脚），仅表单与主文案不同。

## 4. 页面级布局需求

### 4.1 登录页（企业化 / 集团化重做）

登录页承担 **品牌第一印象与组织安全感**，需区别于普通 SaaS 小卡片登录，按 **集团级统一认证入口** 规格设计。

**布局与分区**

- **推荐结构**：**宽幅分栏** —— 左侧 **品牌与价值区**（约占 40%～50% 宽），右侧 **登录表单区**；大屏下表单区仍保持 **最大宽度上限**（如 420～480px），避免输入行过长。
- **品牌区（左）**  
  - **集团 / 企业标识位**：支持 **Logo + 产品全称 + 可选副标题**（如「内网穿透统一管控平台」）；预留 **二级标识**（事业部 / 子公司 / 租户名）展示位，弱对比、小一号字。  
  - **信任与合规**：可选 **一行** 静态文案（如「适用于集团多节点、多业务线统一纳管」），避免堆砌口号。  
  - **视觉底**：低饱和 **大图 / 渐变 / 抽象几何**（与主色蓝同色系延展），**无人物实拍硬性要求**，但需 **专业、克制、无廉价素材感**。  
  - **装饰元素**：可加入 **极淡网格线 / 光晕**，与飞书系 B 端气质一致，忌花哨插画抢焦点。
- **表单区（右）**  
  - **卡片**：白底、圆角、轻边框或轻阴影；内边距 **宽裕**（不少于 32px）。  
  - **标题层级**：主标题「登录」+ 可选副标题「使用分配账号登录管理控制台」。  
  - **字段**：用户名、密码；支持 **显示/隐藏密码** 图标按钮。  
  - **主操作**：主色全宽或主按钮显著；下方可设 **次要链接**（忘记密码占位若业务需要）。  
  - **语言**：语言切换置于 **表单区右上角** 或 **顶栏通栏**，与品牌区不抢位。

**集团化细节**

- **安全暗示**：密码框旁可加 **小字提示**（如「请勿在公共设备保存密码」），灰字即可。  
- **页脚通栏**：**版权、备案占位、技术支持邮箱 / 内部 IT 联系方式** 等区域化信息，左对齐或居中，字号 **12～13px**，颜色 **次要灰**。  
- **注册入口**：若开放注册，与主按钮 **视觉层级拉开**（线框按钮或文字链），避免与「组织正式入口」抢注意力。

**状态与适配**

- **加载**：主按钮 **loading** 态；整页骨架可选。  
- **错误**：表单顶或字段下 **简短错误条**（非弹窗优先），符合企业后台习惯。  
- **窄屏**：品牌区与表单区 **改为上下堆叠**；品牌区可收为 **顶栏条 + 小 Logo**，保证表单仍为一屏核心。

### 4.2 注册页（若开放）

- 与 **4.1** 共用同一套 **左右分栏 / 品牌 + 表单** 栅格与组件样式。  
- 表单字段：用户名、密码、确认密码（建议）、注册协议勾选（若需要合规）。  
- 底部：**返回登录** 明确可点。

### 4.3 仪表盘（集团化工作台）

仪表盘定位为 **「集团穿透服务工作台」首页**，不是简单监控页：需 **一屏看清运行态势 + 快速进入管控**，信息架构与视觉权重按 **决策 / 运维** 双场景优化。

**整体结构（自上而下）**

1. **页头区**  
   - **标题**：如「工作台」或「总览」+ **可选环境标签**（生产 / 测试，胶囊）。  
   - **一句话摘要**（可选）：今日在线客户端数、告警数等 **一行 KPI 摘要**（灰字）。  
   - **快捷入口（Chips 或图标按钮）**：**客户端管理**、**域名解析**、**新建客户端** 等，减少侧栏点击路径。

2. **核心 KPI 带（第一屏强关注）**  
   - **不少于 4 枚、建议 5～6 枚** 指标卡：连接端口、客户端总数、**在线客户端数**、TCP 连接数、**UDP 相关或隧道总数**、**告警 / 离线客户端数（占位）** 等，按产品数据可调整。  
   - 卡片：**统一高度**、数字 **tabular**、同比/环比 **小字占位**（若后续有数据可接）。  
   - **异常态**：离线数 >0 时卡片可用 **弱警告色左边框或角标** 提示（不默认整卡红底）。

3. **运行态势（双列或 Tab）**  
   - **左**：**配置信息**（服务端桥接、端口、版本等）— **键值列表**，可折叠「高级」。  
   - **右**：**系统信息**（CPU、内存、负载、连接、带宽）— **进度条 + 数值**，与现有一致但 **间距加大**。  
   - **可选**：两区块顶部加 **小标题 +「查看详情」** 占位（跳转帮助或监控子页）。

4. **趋势与分布（第二屏，可折叠模块）**  
   - **时间序列**：负载、CPU、内存、带宽等 **折线图**，默认展示 **最近 24h 或当日**（占位文案）。  
   - **结构分布**：隧道类型饼图 / 环图；**客户端分组占比**（树顶级分布，占位）— 与 **客户端分组** 能力叙事一致。  
   - 每图 **独立白卡片**，**标题 + 图例 + 导出占位**（图标即可）。

5. **空数据与加载**  
   - 无隧道 / 无客户端时 **引导卡片**：「创建第一个客户端」「了解分组管理」等，**主按钮 + 次级链**。

**优化方向（功能层产品目标，设计需预留）**

- **告警条**：顶栏下方 **可展开** 的黄色/蓝色 info 条（证书到期、离线节点多等），不占固定高度时可折叠。  
- **权限占位**：若未来分租户 / 分 BU，**标题区** 预留 **当前组织范围** 文案位。

### 4.4 客户端页（左树右表 + 分组）

**布局**

- **整体**：**左侧固定宽度可拖拽**（建议 240～320px，可拖拽加宽）+ **右侧弹性填满**；左右 **分割条可拖拽** 调整比例（桌面端）。  
- **左侧：客户端分组树**  
  - **树形数据**：支持 **无限层级**（至少设计 **3 层** 视觉稿：集团 / 区域 / 项目）；根节点可为「**全部客户端**」或租户名。  
  - **节点内容**：文件夹图标 + **分组名称** + **可选数量角标**（该分组下客户端数）。  
  - **交互**：单击 **选中分组**（高亮背景 + 主色左边条）；**展开/折叠** 三角；**右键菜单占位**（新建子分组、重命名、删除、排序）— 具体以产品为准，设计需 **树行高 ≥ 36px** 便于点击。  
  - **顶栏**：搜索树（过滤节点）、**「新建分组」** 主色或线框按钮。  
  - **空树**：插画 +「创建第一个分组」引导。  
- **右侧：客户端列表**  
  - 工具栏、表格、行展开与 **§8.5** 清单一致；列表数据 **随左侧选中节点过滤**（选中「全部」显示全部，选中子分组仅显示该分组下客户端）。  
  - **新增客户端** 时 **默认归属当前选中分组**（设计稿用 **只读展示当前分组名 + 可改为树形选择器** 体现）。

**分组与数据关系（设计表达）**

- 每个客户端在表单中 **必选或可选「所属分组」**（树形选择器，见 **8.6**）。  
- 支持 **拖拽客户端到树节点**（高级交互，可选一期线框）：**拖放高亮**、**非法拖放提示**。

### 4.5 列表类页面（隧道、域名等；不含客户端主列表）

- **工具栏**：左为 **主操作**（如「新增」）主色按钮；右为 **搜索框**（圆角输入框，可带清除）、筛选若需要可收进下拉。  
- **表格**：斑马纹可选；表头固定；列宽可拖拽或「列显示」设置；行高舒适。  
- **行操作**：图标按钮成组，间距统一；危险操作（删除）用警告色或二次确认，不默认大红铺满。  
- **展开行**：展开区背景略区别于主行，内文分段（流量、命令示例等）用小标题或分割线。

### 4.6 通用表单（隧道、域名等新增 / 编辑）

- **布局**：单列表单为主，标签左对齐或顶对齐；相关字段分组，组间留白或 **分组标题**（**飞书表单分组**：小标题 + 分隔，密度适中）。
- **长表单**：锚点导航或分步向导（可选），避免单页过长无定位。
- **隧道类型切换**：切换类型时字段显隐变化需有 **过渡或简短说明**，避免突兀跳变。
- **底部操作**：主按钮「保存 / 提交」；次要「取消」返回列表，位置固定底栏或表单底部右对齐。

### 4.7 客户端表单（协议能力与企业字段）

客户端新建 / 编辑除 **限速、压缩、密钥** 等基础项外，须体现 **「纳管对象 + 协议能力」** 的集团化配置，避免与隧道表单混淆。

**分组**

- **所属分组**：**树形选择器**（单选叶子或任意节点，产品定）；未选时默认根分组或「未分组」占位；与 **§4.4** 左侧树 **数据同源展示**。

**穿透协议能力（必选设计区块）**

- 独立分组标题，建议：**「支持的穿透类型」** 或 **「协议能力」**。  
- 以 **多选** 或 **多枚开关** 呈现（二选一需统一组件），覆盖当前产品中的 **全部内网穿透协议类型**，至少包括：  
  - **TCP**；**UDP**；**HTTP 代理**；**SOCKS5**；**秘密通道**；**P2P**；**域名解析（HTTP/S）**；**文件服务**（若产品保留）。  
- 每项附 **简短说明**（tooltip 或副文案）：典型用途 1 行即可。  
- **交互**：全选 / 清空（可选）；禁用态需说明原因（权限不足等占位）。  
- **视觉**：与「隧道里选类型」区分 —— 此处表达 **客户端被允许使用的协议范围 / 策略**，可用 **标签组 + 已选数量** 在列表页回显（**§8.5** 增加列或角标占位）。

**说明**

- 若后端策略为「仅展示、实际仍由隧道配置决定」，设计仍保留本区块，并可用 **辅助说明灰字** 标明「与隧道配置联动」类文案（文案由产品定）。

### 4.8 域名解析表单

- 与 **§4.6** 通用表单一致；协议（HTTP/HTTPS/全部）与证书相关字段 **随协议变化显隐**，证书区单独成组。

### 4.9 帮助 / 说明

- 长文阅读：限制最大宽度（约 680～800px），段落与列表间距舒适；代码块使用等宽字体与浅灰背景。

## 6. 响应式与适配

- **断点**：桌面优先；平板缩小侧栏或顶栏收纳菜单；手机端侧栏改为抽屉或底栏（若投入有限，至少保证表格横向滚动 + 关键列冻结方案）。
- **客户端页**：窄屏下 **分组树收入抽屉**（顶部按钮「选择分组」打开），右侧列表全宽；或 **上下布局**（树上表下）。  
- **触摸**：移动端可点击区域与间距加大；**树节点** 展开区域足够大。
